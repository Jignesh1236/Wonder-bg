WonderBG Docs

Zero-dependency animated gradient backgrounds. One script tag, ten presets, full JS API.

v1.3.0 No dependencies ~25 KB MIT

Installation

Copy wonder-bg.js into your project and add a single script tag. No npm, no bundler.

<script src="wonder-bg.js"></script>
💡 Place the script in <head> or just before </body>. Auto-init runs on DOMContentLoaded so either works.

Quick start

The fastest way — add a wb-{preset} class to any element. WonderBG auto-discovers and animates it.

<!-- Step 1: add the script -->
<script src="wonder-bg.js"></script>

<!-- Step 2: add a class -->
<div class="wb-aurora"></div>

The element fills its container. Make sure it has a defined width and height (or set via CSS).

CSS class shortcuts

Any of the 10 preset names work as a class: wb-aurora, wb-sunset, wb-ocean, etc.

<div class="wb-aurora"></div>
<div class="wb-neon"></div>
<div class="wb-sunset"></div>

Data attributes

Use data-wb-background (and optionally data-wb-background-mode) for declarative setup — great on <img> elements.

<!-- gradient (default) -->
<div data-wb-background="ocean"></div>

<!-- tint overlay on a photo -->
<img src="photo.jpg"
     data-wb-background="sunset"
     data-wb-background-mode="tint" />

<!-- grain overlay on a photo -->
<img src="photo.jpg"
     data-wb-background="mono"
     data-wb-background-mode="grain" />

CSS custom properties

Fine-tune speed, blur, grain, and interactivity inline — no JS needed.

<div class="wb-aurora" style="
  --wb-background-speed: 2;
  --wb-background-blur: 40;
  --wb-background-grain: 0.10;
  --wb-background-interactive: true;
"></div>

JavaScript API

WonderBG.create(element, config) → instance

Creates and starts an animation on element. Returns an instance with update() and destroy().

const inst = WonderBG.create(document.getElementById('hero'), {
  preset: 'aurora',
  mode:   'gradient',
  speed:  1.5,
  blur:   50,
  grain:  0.05,
  interactive: true,
  colors: [
    'rgba(32,178,170,0.85)',
    'rgba(0,128,128,0.70)',
    'rgba(22,33,62,0.90)'
  ]
});

instance.update(config)

Live-update any config property. Changes take effect immediately — the animation transitions smoothly.

inst.update({ preset: 'neon', speed: 2 });
inst.update({ interactive: true });
inst.update({ colors: ['rgba(255,0,150,0.8)', 'rgba(0,220,255,0.7)', 'rgba(180,0,255,0.6)'] });

instance.destroy()

Stops the animation, cancels requestAnimationFrame, and removes all injected DOM nodes. Runs automatically if the host element is removed from the DOM.

inst.destroy();

Mode — Gradient

The default mode. Fills the element with animated color blobs. Works on any block-level container.

<!-- HTML -->
<div class="wb-sunset" style="height:200px; border-radius:16px"></div>

<!-- JS -->
const inst = WonderBG.create(el, { preset: 'sunset', mode: 'gradient' });

Mode — Tint on Photo

Gradient blobs composited over an image using mix-blend-mode: soft-light. The image stays visible; the gradient adds warmth and atmosphere.

<img src="photo.jpg"
     data-wb-background="ocean"
     data-wb-background-mode="tint" />

<!-- JS equivalent -->
WonderBG.create(imgEl, { preset: 'ocean', mode: 'tint', blur: 30 });

Mode — Grain on Photo

Film-grain noise overlaid on the photo — no colour blobs, just texture. Good for a cinematic, analog feel.

<img src="photo.jpg"
     data-wb-background="mono"
     data-wb-background-mode="grain" />

Mode — Gradient Text

Use WonderBG's preset colors as the text color via CSS background-clip: text. No canvas — pure CSS gradient derived from the preset's blob colors.

// 1. Pull the preset colors
const colors = WonderBG.presets.neon.blobs.map(b => b.color);

// 2. Build a CSS gradient
const grad = `linear-gradient(135deg, ${colors.join(', ')})`;

// 3. Clip to text
const el = document.querySelector('h1');
el.style.backgroundImage       = grad;
el.style.backgroundSize        = '200% 200%';
el.style.webkitBackgroundClip  = 'text';
el.style.backgroundClip        = 'text';
el.style.color                 = 'transparent';
Animated gradient text

Presets

Ten built-in presets. Each defines a background color, three animated blob definitions, default blur, and grain.

Accessing preset data

// All presets
WonderBG.presets

// One preset's blob colors
WonderBG.presets.aurora.blobs.map(b => b.color)
// → ['rgba(32,178,170,0.85)', 'rgba(0,128,128,0.70)', 'rgba(22,33,62,0.90)']

// Background color
WonderBG.presets.aurora.background  // '#0d1117'

Register a custom preset

WonderBG.registerPreset('candy', {
  background: '#1a001a',
  blobs: [
    { color: 'rgba(255,100,200,0.8)', baseX: 0.3, baseY: 0.4,
      radius: 0.5, orbitX: 0.1, orbitY: 0.08, phase: 0,   speed: 0.6 },
    { color: 'rgba(120,0,255,0.7)',  baseX: 0.7, baseY: 0.6,
      radius: 0.55, orbitX: 0.09, orbitY: 0.1, phase: 2.1, speed: 0.45 },
    { color: 'rgba(255,50,100,0.6)',  baseX: 0.5, baseY: 0.2,
      radius: 0.45, orbitX: 0.12, orbitY: 0.06, phase: 4.2, speed: 0.5 }
  ],
  grain: 0.05,
  blur:  60
});

// Then use it like any built-in preset
WonderBG.create(el, { preset: 'candy' });

Performance

  • One <canvas> per element, rendered via requestAnimationFrame.
  • Auto-pauses when element scrolls off-screen (IntersectionObserver).
  • Auto-destroys when element is removed from DOM (MutationObserver).
  • Respects prefers-reduced-motion — pauses when the user prefers reduced motion.
  • Blur GPU-capped: ≤ 35 px on mobile, ≤ 70 px on desktop.

TypeScript

Include wonder-bg.d.ts in your project for full type support.

// tsconfig.json
{
  "include": ["src", "wonder-bg.d.ts"]
}

// usage.ts
import type { WonderBGConfig, WonderBGInstance } from './wonder-bg.d.ts';

const cfg: WonderBGConfig = { preset: 'ocean', speed: 1.2 };
const inst: WonderBGInstance = WonderBG.create(el, cfg);

Browser support: Chrome 80+, Firefox 80+, Safari 14+, Edge 80+. Requires: Canvas 2D, mix-blend-mode, IntersectionObserver.
